character/contacts

A container for the contacts ESI endpoints. You should not require this module directly, as it technically returns a factory function that requires an internal API. Instead an instance is automatically exposed when the module:character is loaded and configured.

(static) add(characterId, contacts, standing, label, accessToken) → {external:Promise}

Source:
  • character/contacts.js, line 115
See:
  • https://esi.tech.ccp.is/latest/#!/Contacts/post_characters_character_id_contacts

Bulk add contacts with the same settings to the given character's contact list via the ESI end point. This makes an HTTP POST request to /characters/{characterId}/contacts. The request is returned as an asynchronous Promise that resolves to an array from the response JSON model. An example return value looks like:

[
  123,
  456
]

The added contacts are unwatched. This promisifies a call to the generated ContactsApi.postCharactersCharacterIdContacts.

Parameters:
Name Type Description
characterId Integer

The character id
contacts Array.<Integer>

Array of contact character ids to add
standing Float

The standing for all contacts
label Integer

Optional; custom label id to add contacts to
accessToken String

Optional; the access token to authenticate contact access of the sending character. If not provided, the default access token is used. This will fail if neither is available.
Returns:

A Promise that resolves to the response of the request

(static) get(id, page, accessToken) → {external:Promise}

Source:
  • character/contacts.js, line 49
See:
  • https://esi.tech.ccp.is/latest/#!/Contacts/get_characters_character_id_contacts

Get a page of contacts for the given character via the ESI end point. The page parameter controls which page is selected. This makes an HTTP GET request to /characters/{id}/contacts. The request is returned as an asynchronous Promise that resolves to an array parsed from the response JSON model. An example value looks like:

[
  {
    "contact_id": 123,
    "contact_type": "character",
    "is_blocked": false,
    "is_watched": true,
    "standing": 10
  }
]

This promisifies a call to the generated ContactsApi.getCharactersCharacterIdContacts.

Parameters:
Name Type Description
id Integer

The character id
page Integer

The page to fetch, defaults to 1
accessToken String

Optional; the access token to authenticate contact access of the sending character. If not provided, the default access token is used. This will fail if neither is available.
Returns:

A Promise that resolves to the response of the request

Source:

See:

Bulk update contacts to the same settings in the given character's contact list via the ESI end point. This makes an HTTP PUT request to /characters/{characterId}/contacts. The request is returned as an asynchronous Promise that resolves to an empty object on success. The updated contacts are set to unwatched. This promisifies a call to the generated ContactsApi.putCharactersCharacterIdContacts.
