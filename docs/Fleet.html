Fleet

Fleet

An api adapter over the end points handling a character's fleet via functions in the fleets ESI endpoints. You should not usually instantiate this directly as its constructor requires an internal api instance.

Constructor

new Fleet(api, token, fleetId)

Source:
  • api/character/Fleet.js, line 275

Create a new Fleet function using the given api with SSO access from token. The fleetId must also be provided.

Parameters:
Name Type Description

The api provider
token String

The SSO access token for the character
fleetId Number

The fleet id of the fleet the character belongs to

Methods

info() → {Promise}

Source:
  • api/character/Fleet.js, line 321

Get fleet details from the ESI endpoint. This makes an HTTP GET request to /fleets/{id}/. The request is returned as an asynchronous Promise that resolves to an object parsed from the response JSON model. An example value looks like:

{
  "is_free_move": false,
  "is_registered": false,
  "is_voice_enabled": false,
  "motd": "This is an <b>awesome</b> fleet!"
}

This promisifies a call to the generated FleetsApi.getFleetsFleetId.

Returns:

A Promise that resolves to the response of the request

invite(invitation) → {Promise}

Source:
  • api/character/Fleet.js, line 378

Send a fleet invite to a character with the ESI endpoint. This makes an HTTP POST request to /fleets/{id}/members/. The request is returned as an asynchronous Promise that resolves to an empty object on success. An example invitation value looks like:

{
  "character_id": 0,
  "role": "fleet_commander",
  "squad_id": 0,
  "wing_id": 0
}

This promisifies a call to the generated FleetsApi.postFleetsFleetIdMembers.

    "takes_fleet_warp": true,
    "wing_id": 2073711261968
  }

move(memberId, moveOrder) → {Promise}

Source:
  • api/character/Fleet.js, line 419

Move or update a member in a fleet with the ESI endpoint. This makes an HTTP PUT request to /fleets/{fleetId}/members/{memberId}/. The request is returned as an asynchronous Promise that resolves to an empty object on success. An example move order looks like:

{
  "role": "fleet_commander",
  "squad_id": 0,
  "wing_id": 0
}

This promisifies a call to the generated FleetsApi.putFleetsFleetIdMembersMemberId.
