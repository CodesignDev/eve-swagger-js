War

War

An api adapter that provides functions for accessing various details for an war specified by id, via functions in the wars ESI endpoints. You should not usually instantiate this directly as its constructor requires an internal api instance.

Constructor

new War(api, warId)

Source:
  • api/Wars.js, line 14

Create a new War for the given api provider and specific warId.

Parameters:
Name Type Description
api ApiProvider

The api provider used to generate web requests
warId Number

The war id that is used for all requests

Methods

info() → {Promise}

Source:
  • api/Wars.js, line 61

Get information on a war from the ESI endpoint. This makes an HTTP GET request to /wars/{id}/. The request is returned as an asynchronous Promise that resolves to an array parsed from the response JSON model. An example value looks like:

{
  "aggressor": {
    "corporation_id": 986665792,
    "isk_destroyed": 0,
    "ships_killed": 0
  },
  "declared": "2004-05-22T05:20:00Z",
  "defender": {
    "corporation_id": 1001562011,
    "isk_destroyed": 0,
    "ships_killed": 0
  },
  "id": 1941,
  "mutual": false,
  "open_for_allies": false
}

This promisifies a call to the generated WarsApi.getWarsWarId.

Returns:

A Promise that resolves to the response of the request

Type
Promise

killmails(page) → {Promise}

Source:
  • api/Wars.js, line 163
See:
  • Killmail#get

Get killmail ids and hashes from a war from the ESI endpoint. This makes an HTTP GET request to /wars/{id}/killmails/. The request is returned as an asynchronous Promise that resolves to an array parsed from the response JSON model. An example value looks like:

[
  {
    "killmail_hash": "8eef5e8fb6b88fe3407c489df33822b2e3b57a5e",
    "killmail_id": 2
  },
  {
    "killmail_hash": "b41ccb498ece33d64019f64c0db392aa3aa701fb",
    "killmail_id": 1
  }
]

This promisifies a call to the generated WarsApi.getWarsWarIdKillmails.

Parameters:
Name Type Default Description
page Number 0

Optional; the page of killmails to fetch, starting with page 1. If not provided then all mails are returned.
Returns:

A Promise that resolves to the response of the request

Type
Promise
