Module: character/mail

A container for the mail ESI endpoints. You should not require this module directly, as it technically returns a factory function that requires an internal API. Instead an instance is automatically exposed when the module:character is loaded and configured.

(static) getInbox(id, labelIds, lastMailId, accessToken) → {external:Promise}

Get the 50 most recent mail headers in the characters inbox. If labelIds is present, then only mail within those labels are returned. If lastMailId is provided, up to the 50 mails just prior to that mail id will be returned, allowing you to paginate backwards.

This makes an HTTP GET request to /characters/{id}/mail/. The request is returned as an asynchronous Promise that resolves to an array parsed from the response JSON model. An example value looks like:

[
  {
    "from": 90000001,
    "is_read": true,
    "labels": [
      3
    ],
    "mail_id": 7,
    "recipients": [
      {
        "recipient_id": 90000002,
        "recipient_type": "character"
      }
    ],
    "subject": "Title for EVE Mail",
    "timestamp": "2015-09-30T16:07:00Z"
   }
]

This promisifies a call to the generated MailApi.getCharactersCharacterIdMail.

Parameters:
Name Type Description
id Integer

The character whose mail is fetched
labelIds Array.<Integer>

The label ids to restrict returned mail to, or []|null|undefined for no restriction
lastMailId Integer

Optional mail id to constrain returned mail to have come prior to the given mail id
accessToken String

Optional access token to authenticate the request, which overrides any default access token of the factory. If neither this nor the default token are provided, this will fail.
