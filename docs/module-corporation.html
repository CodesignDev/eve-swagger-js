corporation

A container for the corporation ESI endpoints. You should not require this module directly, as it technically returns a factory function that requires an internal API. Instead an instance is automatically exposed when the module:eve_swagger_interface is loaded and configured.

(static) get(id) → {external:Promise}

Source:
  • corporation.js, line 41
See:
  • https://esi.tech.ccp.is/latest/#!/Corporation/get_corporations_corporation_id

Get a corporation's public info from the ESI endpoint. This makes an HTTP GET request to corporations/{id}/. The request is returned as an asynchronous Promise that resolves to an object parsed from the response JSON model. An example value looks like:

{
  "alliance_id": 434243723,
  "ceo_id": 180548812,
  "corporation_name": "C C P",
  "member_count": 656,
  "ticker": "-CCP-"
}

This promisifies a call to the generated CorporationApi.getCorporationsCorporationId.

Source:

See:

Get a corporation's member list with roles for each character from the ESI endpoint. This makes an HTTP GET request to corporations/{id}/roles/. The request is returned as an asynchronous Promise that resolves to an array parsed from the response JSON model. An example value looks like:

[
  {
    "character_id": 1000171,
    "roles": [
      "Director",
      "Station_Manager"
    ]
  }
]

This promisifies a call to the generated CorporationApi.getCorporationsCorporationIdRoles.

Parameters:
Name Type Description
id Integer

The corporation id
accessToken String

Optional; the SSO access token of a member of the corporation that has the personnel manager or any other grantable role, used to authenticate the request. If not provided, the default access token provided to the factory is used; if that was not set then this request will fail.
Returns:

A Promise that resolves to the response of the request
