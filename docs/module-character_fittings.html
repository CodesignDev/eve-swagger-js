character/fittings

A container for the fittings ESI endpoints. You should not require this module directly, as it technically returns a factory function that requires an internal API. Instead an instance is automatically exposed when the module:character is loaded and configured.

(static) add(characterId, fitting, accessToken) → {external:Promise}

Source:
  • character/fittings.js, line 122
See:
  • https://esi.tech.ccp.is/latest/#!/Fittings/post_characters_character_id_fittings

Create a new fitting for the charactervia the ESI end point. This makes an HTTP POST request to /characters/{characterId}/fittings/. The request is returned as an asynchronous Promise that resolves to an object containing the created fit's id. An example return value looks like:

{
  "fitting_id": 2
}

The fitting is described by an object containing all items, an example being:

{
  "description": "string",
  "items": [
    {
      "flag": 0,
      "quantity": 0,
      "type_id": 0
    }
  ],
  "name": "string",
  "ship_type_id": 0
}

This promisifies a call to the generated FittingsApi.postCharactersCharacterIdFittings.

Parameters:
Name Type Description
characterId Integer

The character id
fitting Object

The fitting specification
accessToken String

Optional; the access token to authenticate contact access of the sending character. If not provided, the default access token is used. This will fail if neither is available.
Returns:

A Promise that resolves to the response of the request

(static) getAll(id, accessToken) → {external:Promise}

Source:
  • character/fittings.js, line 53
See:
  • https://esi.tech.ccp.is/latest/#!/Fittings/get_characters_character_id_fittings

Get all fittings for the given character via the ESI end point. This makes an HTTP GET request to /characters/{id}/fittings/. The request is returned as an asynchronous Promise that resolves to an array parsed from the response JSON model. An example value looks like:

[
  {
    "description": "Awesome Vindi fitting",
    "fitting_id": 1,
    "items": [
      {
        "flag": 12,
        "quantity": 1,
        "type_id": 1234
      }
    ],
    "name": "Best Vindicator",
    "ship_type_id": 123
  }
]

This promisifies a call to the generated FittingsApi.getCharactersCharacterIdFittings.

Source:

Remove the fitting by its id from the given character's list via the ESI end point. This makes an HTTP DELETE request to /characters/{characterId}/fittings/{fittingId}/. The request is returned as an asynchronous Promise that resolves to an empty object from the response JSON model. This promisifies a call to the generated FittingsApi.deleteCharactersCharacterIdFittingsFittingId.
