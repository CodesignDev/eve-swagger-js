Module: killmails

A container for the killmails ESI endpoints. You should not require this module directly, as it technically returns a factory function that requires an internal API. Instead an instance is automatically exposed when the module:eve_swagger_interface is loaded and configured.

Methods

(static) get(id, hash) → {external:Promise}

Get the details from a killmail with the ESI endpoint. This makes an HTTP GET request to /killmails/{id}/{hash}/. The request is returned as an asynchronous Promise that resolves to an object parsed from the response JSON model. An example value looks like:

  "attackers": [
    {
      "character_id": 95810944,
      "corporation_id": 1000179,
      "damage_done": 5745,
      "faction_id": 500003,
      "final_blow": true,
      "security_status": -0.3,
      "ship_type_id": 17841,
      "weapon_type_id": 3074
    }
  ],
  "killmail_id": 56733821,
  "killmail_time": "2016-10-22T17:13:36Z",
  "solar_system_id": 30002976,
  "victim": {
    "alliance_id": 621338554,
    "character_id": 92796241,
    "corporation_id": 841363671,
    "damage_taken": 5745,
    "items": [
      {
        "flag": 20,
        "item_type_id": 5973,
        "quantity_dropped": 1,
       "singleton": 0
      }
    ],

This promisifies a call to the generated KillmailsApi.getKillmailsKillmailIdKillmailHash.
