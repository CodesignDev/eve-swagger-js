Module: character/calendar

A container for the calendar ESI endpoints. You should not require this module directly, as it technically returns a factory function that requires an internal API. Instead an instance is automatically exposed when the module:character is loaded and configured.

(static) get(characterId, accessToken, fromEventId) → {external:Promise}

Get up to 50 future calendar event summaries for the given character via the ESI end point. If fromEventId is provided, the returned events will be the next 50 events chronologically beyond the event.

This makes an HTTP GET request to /characters/{characterId}/calendar. The request is returned as an asynchronous Promise that resolves to an array parsed from the response JSON model. An example value looks like:

[
  {
    "event_date": "2016-06-26T20:00:00Z",
    "event_id": 1386435,
    "event_response": "accepted",
    "importance": 0,
    "title": "o7 The EVE Online Show"
  }
]

This promisifies a call to the generated CalendarApi.getCharactersCharacterIdCalendar.

Parameters:
Name Type Description
characterId Integer

The character id
accessToken String

Optional; the access token to authenticate contact access of the sending character. If not provided, the default access token is used. This will fail if neither is available.
fromEventId Integer

Optional; the event id that returned events will be chronologically later than.

(static) getEvent(characterId, eventId, accessToken) → {external:Promise}

Get all information for a specific event via the ESI end point. This makes an HTTP GET request to /characters/{characterId}/calendar/{eventId}. The request is returned as an asynchronous Promise that resolves to an object parsed from the response JSON model. An example value looks like:

{
  "date": "2016-06-26T21:00:00Z",
  "duration": 60,
  "event_id": 1386435,
  "importance": 1,
  "owner_id": 1,
  "owner_name": "EVE System",
  "owner_type": "eve_server",
  "response": "Undecided",
  "text": "o7: The EVE Online Show features latest developer news, fast paced action, community overviews and a lot more with CCP Guard and CCP Mimic. Join the thrilling o7 live broadcast at 20:00 EVE time (=UTC) on <a href=\"http://www.twitch.tv/ccp\">EVE TV</a>. Don't miss it!",
  "title": "o7 The EVE Online Show"
}

(static) updateResponse(characterId, eventId, response, accessToken) → {external:Promise}

Update response for a specific event via the ESI end point. This makes an HTTP PUT request to /characters/{characterId}/calendar/{eventId}. The request is returned as an asynchronous Promise that resolves to an empty object on success. Allowed response values can be

  1. 'rejected'
  2. 'accepted'
  3. 'undecided' This promisifies a call to the generated CalendarApi.putCharactersCharacterIdCalendarEventId.
